Introduction

This project was started with clean development, SEO and performance in mind.

  • First of all, we wanted to separate Bootstrap 3 code from our HTML source to get rid of the mix-up of data with presentation code.
  • Secondly, we wanted our data to be semantically meaningful and easy to digest by search engines.
  • Third, we wanted it all to be really fast.

The result is shown in this website. The HTML source code consists of only microdata from Schema.org, while the resulting DOM view is rendered with client side Mustache templates based on Bootstrap 3. It is not just really speedy, but it serves your HTML to search engines just the way they like it.

What's in the box?

Before we start: we assume that you have some background in working with Jquery and Bootstrap 3. It also helps if you know things like AMD - but you can figure that our from the code, too.

The Paradax source code on Github contains the following libs. It helps if you know what they do.

  • Jquery - version 1.10.2
  • Bootstrap 3 - customized version with fewer components
  • MicrodataJS - stripped down version for speed
  • Mustache
  • RequireJs
  • Hogan

Furthermore, there is plug-in for Hogan and requireJs, a RequireJs optimizer file and even a Node.js build script. But you can just ignore most of that.

HTML with microdata

Microdata is a syntax for embedding machine-readable metadata in HTML. It intends to make HTML semantically meaningful. Specifically, the microdata as proposed on Schema.org can be used by developers to markup their pages in ways recognized by major search providers. The project is backed by Google, Yandex, Bing, Yahoo and more.

Schema.org example

This snippet shows an example of data formatting for the documentation page that you are reading right now. Every chapter is laid out like this without any further structure. View the source code and you will see.

<div itemscope itemtype="http://schema.org/TechArticle">
  <h2 itemprop="headline">
    Fast food for search engines
  </h2>
  <div itemprop="articleBody">
      Bla bla bla
  </div>
</div>

This is a very simple example, but you can do this for people, organizations, web applications and so on. Browse through this website and view the source for examples.

Mustache templates

Search engines devour microdata easily, but we need its contents also to appear on our web page. Luckily, there is help. MicrodataJS is a jQuery plugin that provides an API for accessing microdata.

JSON microdata example

It serves JSON like this.

{
  "items": [
    {
      "type": [
        "http://schema.org/TechArticle"
      ],
      "properties": {
        "name": [
          "someName"
        ],
        "headline": [
          "Fast food for search engines"
        ],
        "articleBody": [
          "Bla bla bla"
        ]
      }
    }
  ]
}

This stuff is easy to use in client side templates with some Javascript. Mustache is a great choice for client side templates, but there are many more options. However, we like it because it is really simple.

Bootstrap 3 Mustache example with microdata

The following could render a number of chapters on this documentation page wrapped in a Bootstrap row. 

<div class="container">
  <div class="row">
    <div class="col-md-9 col-sm-8">
      {{#items}}
        {{#TechArticle}}
          {{#properties}}
            <a id="{{#name}}{{{.}}}{{/name}}" />
            <h2>{{#headline}}{{{.}}}{{/headline}}</h2>
            {{#articleBody}}{{{.}}}{{/articleBody}}
    	  {{/properties}}
        {{/TechArticle}}
      {{/items}}
  </div>
</div>

Mustache is so simple, that it is not possible to use a conditional statement based on the value of a variable. You can only detect if it exists or not. That is why I added the name of each itemType in the microdata JSON as a variable that is set to true if it exists.

Now, you can use the inspector in Chrome or Firefox to see the DOM of any of the Paradax pages - it's all Bootstrap 3.

Optimization

At Peecho, we use the Twitter Hogan compiler to compile Mustache templates. It really speeds up rendering, especially if you pre-compile your templates on the server. The Paradax build script does that for you.

The RequireJs optimizer script wraps all Javascript in a single file, containing all Mustache templates and partials in their compiled state. After initial load, the entire presentation code is cached in your browser.

While rendering the templates, we hide the view to prevent reflow. This is a web browser process for re-calculating the positions and geometries of elements while rendering the view. Once the rendering is done, you can display the content all at once.

If you are really going for it, you can serve all of it from a CDN. We use Google PageSpeed service and Cloudfront at Peecho. Both are awesome.
